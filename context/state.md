# Active

- Renovate currently authenticates as the ordinary `DevGuyRash` user rather than a dedicated bot or GitHub App. The automerge gate binds that exact immutable user ID, same-repository ID, and `Renovate Bot <bot@lablabland.com>` commit identity, but a manually authored PR using the same user credentials can reproduce those attributes. WHEN dedicated Renovate credentials are available THEN you SHALL replace the shared-user identity in `.github/automerge-policy.json` with the bot/App identity before treating author verification as cryptographic separation.
- Private-repository branch protections and rulesets remain unavailable on the current GitHub Free organization plan. The automerge sweep fails closed on its own required-check policy, but direct pushes and human merges cannot be forced through those checks until the plan or repository visibility changes.
- Cross-repository adoption and root skill-projection jobs still need a `SECURITY_AUDIT_TOKEN` with read access to the declared private repositories. WHEN configuring that credential THEN you SHALL add it to both `FutureDevGuys/.github` and `FutureDevGuys/personal-containers` before treating the scheduled audit or hosted projection job as operational.
